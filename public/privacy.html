Privacy Policy

Effective date: April 17, 2026.

Autopilot-AI is built to help users organize work from connected services. This policy describes what information the product processes, why it is used, and the controls users have over that data.

Information we process

  • Account details such as email address and authentication identifiers.
  • Connected workspace data that a user explicitly authorizes, such as Gmail and Google Calendar data.
  • Manual tasks, notes, calendar blocks, and workspace preferences a user enters inside the product.
  • Operational metadata needed to keep the service working, such as sync timestamps and error logs.

How information is used

  • To authenticate users and keep their workspace session available.
  • To read authorized work sources and generate action-item planning output requested by the user.
  • To show calendar context, editable user-created blocks, and productivity views inside the app.
  • To improve service reliability, prevent abuse, and troubleshoot sync or planning errors.

Google user data

Read-only by default

Connected Gmail and Google Calendar access is requested only after the user signs in and explicitly starts the connection flow.

Purpose limitation

Google data is used to provide user-requested productivity features such as reading Gmail message metadata and snippets, identifying actionable threads, and reflecting calendar context in the daily plan.

Server-side connection storage

Autopilot-AI stores provider connection tokens behind server-side controls so the user does not need to reconnect Google every time the workspace syncs.

No sale of Google data

Autopilot-AI does not sell Google user data or use it for advertising.

Google API policy

Use of data received from Google APIs is intended to stay aligned with the Google API Services User Data Policy, including Limited Use requirements.

How Google Workspace data is handled in the current product

  • Recent Gmail messages are synced using sender, subject, labels, received time, and snippet-level preview data used for task extraction and ranking.
  • Google Calendar events are synced to reflect meeting timing, availability, and scheduling context inside the planner.
  • Connected-provider tokens are stored server-side with encryption-oriented controls so authorized syncing can continue after sign-in.
  • When a user explicitly runs AI planning, synced email and calendar context can be processed by the configured model provider to generate the requested plan output.
  • Google Workspace data is used for user-facing productivity features, not for advertising or sale of user data.

Sharing, storage, and retention

  • Data is shared only with infrastructure and service providers needed to operate the product.
  • Connected-provider tokens and privileged integration workflows are intended to remain behind server-side controls.
  • Users can disconnect providers, sign out, and stop future syncing.
  • Information is retained only as long as needed to operate the service, comply with legal obligations, and maintain user-requested workspace history.

User controls

  • Users can choose whether to sign in with Google or email.
  • Users can decide whether to connect Google Workspace after sign-in.
  • Users can review connected-source status from inside the app before syncing data.
  • Users can remove their local workspace settings and disconnect providers at any time.
